# 简单工厂模式


简单工厂模式（Simple Factory Pattern）实际上并不属于GoF（Gang of Four，四人帮）在《设计模式：可复用面向对象软件的基础》一书中定义的23种经典设计模式之一。它是一种常用的设计模式，但更像是一种编程习惯或技巧，而不是正式的设计模式。

尽管如此，简单工厂模式通常被认为是工厂方法模式（Factory Method Pattern）和抽象工厂模式（Abstract Factory Pattern）的一个简化版本，因此它可以被归类为创建型模式（Creational Patterns）的一种。

简单工厂模式通过一个单一的工厂类来创建不同类型的对象，客户端只需要传递一个参数来指定要创建的对象类型，而不需要直接实例化对象。这种模式在实际开发中非常常见，尤其是在需要根据条件创建不同类型对象的场景中。
